# 中医药试卷出卷

## 用途

用于分析用户提供的中医药考研、医师资格考试、实践技能、课程期末或练习试卷，并据此编制**原创**试卷、答案解析、评分细则和质量审查报告。默认中文输出。

先读取 `references/corpus-observations.md`。该文件只提供本地样本的结构观察，**不是官方考试规范**。凡涉及题型、题量、分值、站点或报名/考试规则，优先使用用户提供的现行官方大纲、考试说明或院校招生说明；缺失时必须明确写出假设，不得声称“官方格式”。

## 先判定试卷类型

| 类型 | 优先结构 |
|---|---|
| 中医药考研/中医综合 | 客观题为主；可使用单项最佳选择、共用备选答案/题组、病例串联题。重点考查经典与基础理论、诊断、方药、针灸、临床辨治的迁移。 |
| 医师资格理论或考证练习 | 按用户给定大纲和题型配比；可混合单选、多选、填空、名词解释、简答和病例分析。 |
| 医师资格实践技能 | 使用站点式任务：病例采集/分析、基本操作、临床答辩；每题必须有任务、观察点、评分点、扣分条件和安全边界。 |
| 课程期末 | 以课程教学目标和授课范围为第一约束；题型可由基础识记、理解应用和综合病例组成。 |
| 题库/练习 | 先建立知识点标签、难度、认知层级、答案和解析，再按蓝图抽题或改编。 |

若用户同时给出“考研”和“考证”，必须分开生成蓝图，不可混用：前者强调学科综合与病例推理，后者可包含临床操作、病史采集与答辩。

## 工作流

### 1. 收集约束并声明假设

至少确定：考试类型、适用大纲/范围、受众水平、时长、总分、题量、题型、知识模块、难度、是否需要答案解析与评分细则。

信息缺失时，先给出“拟定蓝图 + 假设清单”，并将下列字段标记为 `待确认`：题量、分值、考试时长、官方题型、覆盖范围。

建立来源清单并标记：`official`、`user-provided-unverified` 或 `practice-sample`。只从样本学习题目结构和能力要求；不得复制原题题干、选项、病例、答案或解析。

### 2. 先出试卷蓝图，后写题

使用 `assets/paper-blueprint-template.md`。蓝图必须满足：

- 知识模块与用户大纲一一对应；
- 标明每个模块的题量、分值、认知层级、难度与题型；
- 分值、题量、时长可加总；
- 病例题明确考查链条：`病名/诊断 -> 证候/病机 -> 治法 -> 方药或针灸/鉴别/处置`；
- 实践技能题明确考生任务、所需物品、可观察动作、评分点和安全性扣分；
- 同一知识点不要以同一形式重复考查，除非用户要求螺旋复现。

难度默认可采用：易 20% / 中 60% / 难 20%。若用户另有规范，以用户规范为准。难题应体现信息整合、辨证鉴别或病例迁移，不能靠冷僻知识或歧义措辞制造难度。

### 3. 生成原创试卷

依次输出：

1. `试卷说明与蓝图`
2. `试卷正文`（默认不在题干后显示答案）
3. `参考答案与解析`（单独成节/单独文件）
4. `评分细则`（主观题、病例题、站点题必需）
5. `质量审查报告`

#### 客观题规则

- 单项题只允许一个最佳答案；选项同质、长度尽量平衡，避免“以上都对/以上都不对”。
- 共用备选答案或题组题要避免线索互相泄露。
- 病例题保留足够但不过量的四诊、病程、诱因和必要检查信息；题问只考一个清晰判断层级。
- 不用真实患者可识别信息；病例为虚构教学场景。

#### 主观题规则

- 动词明确：`列举`、`辨析`、`说明依据`、`拟定治法`、`比较`。
- 评分点写成可检查的关键词或逻辑节点，标明分值，不以文采计分。
- 避免把有争议的学术观点写成唯一标准答案；有流派差异时说明采用的教材/大纲口径。

#### 实践技能题规则

使用 `assets/skills-station-template.md`。每道站点题必须写明：

- 情境与考生任务；
- 限时、物品或模拟条件；
- 评分点与分值；
- 严重失误/安全性扣分；
- 考官追问与标准要点；
- 教学用途声明：仅用于能力评估，不能替代临床诊疗决策或官方考核文件。

### 4. 质量审查（发布前必做）

逐项检查：

- 覆盖率：每个指定模块均被考查，且比例与蓝图一致；
- 算术：题号连续、题量/分值/时长加总正确；
- 唯一性：单选题只有一个最佳答案，多选题答案边界明确；
- 可答性：题干信息足够，未引入未教概念或隐含前提；
- 术语：病名、证候、治法、方剂、腧穴与用户指定教材口径一致；
- 安全性：不输出面向真实患者的个体化处方、剂量或替代就医建议；
- 原创性：不复刻用户样卷中的连续题干、选项组合、病例数字或解析表述；
- 来源：没有现行官方依据时，不把样卷观察称为官方格式。

## 输出格式

先给“蓝图与假设”，用户确认后再给完整试卷。若用户要求一次性完成，也要在开头列出采用的假设和来源状态。

对“分析试卷”请求，输出：`试卷类别 -> 题型结构 -> 知识模块 -> 能力层级 -> 难度分布 -> 命题特点 -> 来源/质量限制 -> 可复用蓝图`。
