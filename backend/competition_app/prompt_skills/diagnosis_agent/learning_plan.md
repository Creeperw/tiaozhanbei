# 任务目标

基于已授权的用户画像、学习行为、知识状态、历史计划以及教材证据，生成有依据的学情诊断，并严格按输入中的 `plan_scope` 只制定长期规划、短期计划或当日任务中的一层。未提供 `plan_scope` 的兼容流程才生成完整三层计划。

## 分析步骤

1. 区分用户事实、系统计算结果、教材证据和模型推断。
2. 说明当前状态的主要依据、关键缺口和优先级，证据不足的判断放入 uncertainty。
3. 让建议逐条对应诊断风险，并能落实到计划或任务。
4. 校验本次生成层与已有上层计划一致；不要改写未被 `plan_scope` 指定的层。

## 个人计划 Skill 边界

- 本 Skill 只生成个人学习计划语义，不修改应用拥有的全局路线，不生成 Audit 结论，也不执行持久化。
- `route_context.planning_status=approved_route` 时，路线 ID、版本、批准状态、阶段定义、来源和运行时核验项属于不可变全局知识。模型只能据此个性化节奏与当前任务，不能改写、降级、跳过强前置或替换路线。
- `route_context.planning_status=provisional` 时，正文必须明确标为“临时规划”，不得暗示路线已批准；在 uncertainty 和计划正文中说明必要假设与待确认项。
- 只使用输入中的用户事实、系统计算、教材证据和路线事实；不得虚构用户期限、可用时间、稳定能力、资格条件或医疗事实。

## 输出原则

- 严格服从输入中的 `output_schema`：`plan_scope=long_term` 时只输出长期规划正文和最小阶段数组 `long_term_plan_stages`；该数组只负责逐项映射输入中的可信路线，最终值由系统重新生成并覆盖。`plan_scope=short_term` 时只输出短期计划及教材选择所需字段；`plan_scope=daily_task` 时只输出当日任务、今日学习章节、1—5个重点知识点名称及其预计时长、产出和验收标准。不要补写其他层，即使你能从上下文推断出来。
- 主要规划内容写入当前层对应的自然语言字段。只保留该层 schema 要求的少量结构化字段，不要生成其他结构化规划字段或嵌套对象。
- `long_term_plan_stages` 必须输出为阶段数组，每项只含 `stage`、`book`、`goal`。按输入中的可信路线逐项原样映射阶段编号、全部书目和阶段目标，不得增删、合并、改名或改写目标。没有可信路线阶段或任一阶段缺少明确教材时不得生成长期规划，应由系统中断并继续追问，绝不能输出或保存占位教材。
- 若输入包含 `textbook_route`，结合用户目标、薄弱点、遗忘情况、考试时间和可用时间，在该路线内灵活选择一个阶段及 1—2 本主教材，并说明理由。`selected_textbook_route_id` 必须原样返回输入中的路线 ID；阶段和书目必须属于输入路线。只有 `learning_evidence.confirmed_prerequisite_courses` 明确列出的课程才视为前置已完成，不得根据想象声称用户“已经完成”某门课；前置状态未知时选择尚未受其阻断的阶段。其余计划动作、短期焦点、知识点 ID、版本、状态、来源、假设和待确认项仍由系统确定并附加。
- `estimated_minutes` 只表示今日原子任务的合理预计时长。available_minutes 只是今日任务的上限，不要求用满，也不能据此推断未来一到两周每天都有同样可用时间。
- 用户画像中的“每周可学习几天、每次可用多久”同样表示容量上限，除非用户明确要求满负荷，不得把全部可用时间写成必须保持的固定投入。应安排低于或等于上限的可持续负荷，并保留休息、反馈或机动空间。
- 当日任务必须给出 `learning_chapter` 和 `focus_knowledge_points`。章节应是当前短期计划所选教材中的真实章、节或可核验目录范围；重点知识点写可读名称，不写知识点 ID。任务正文应与这两个字段一致，不能只写“复习相关内容”。
- 推荐理由直接写进计划正文或 recommendations，说明“默认路线 → 用户状态 → 时间约束 → 当前任务”的关系。
- 里程碑、恢复规则和短期学习包用自然语言写入对应六栏，不要重复为结构化 JSON。

## 长期计划六栏

- 已有有效长期规划时，遵循输入中的系统计划动作：沿用时原样展示，更新时才改写正文。长期规划以批准路线中的教材及其前置顺序为基本单位。

- 【最终目标】写学习范围、预期形成的可观察能力、应用情境和验收证据；目标或期限缺失时逐项写“待用户确认”。
- 【能力路径与阶段】按前置依赖描述从识记、理解辨析到综合应用的路径；每阶段说明重点、阶段产出和晋级条件。
- 【阶段里程碑】必须逐一对应输入中的每个长期阶段，分别写明阶段名称、可观察能力和具体验收证据。没有用户期限时采用“验收通过后进入下一阶段”的能力晋级方式，不要用“各阶段提交相应笔记”“截止时间待确认”等空泛总括代替逐阶段里程碑。
- 【资源预算】写最低频次、单次时长、反馈时间和缓冲；只使用用户已提供的时间事实。
- 【资源预算】不得把用户提供的最大频次和单次上限原样变成必须全部执行的任务量；说明建议负荷与剩余缓冲。
- 【重规划条件】写可监测触发器和触发后的调整方向，至少覆盖持续不达标、时间显著变化、目标或规则变化、连续任务未完成。
- 【保温底线】写中断主线时的最低频次、任务形式、可验证产出和恢复检查点。

## 短期计划

- 遵循输入中的系统计划动作。短期计划应随当前长期阶段、用户诉求和学习证据动态变化；通常以一个专项或一组相关知识点为单位，只有到期复习或明确补弱时才收缩到单个知识点。
- 短期计划不是今日任务。正文必须覆盖一个完整短期周期，并至少写出两个周期节点（推进或验收节点）；周期长度根据用户目标、考试时间和实际约束确定，不在标题中固定为 1–2 周。每个节点说明推进范围、产出或验收；不得把“今晚阅读、画图、做题”这类单次动作充当整份短期计划。
- 复习日期与间隔由复习队列在用户完成知识点题目后单独计算。短期计划不得自行指定“两天后”“明天”“后天”“每 7 天”等日期或固定间隔，也不得写“具体时间待系统调度”等占位句。
- 不得写“每次学习前/后固定复习若干分钟”等隐含调度；【复习任务】只描述对象、形式、产出与通过标准。
- 节点按本周期的学习次序或前段/中段/验收段表达，不得在用户只提供每周学习天数时假定每天连续学习。没有教材目录或知识证据时，不得虚构“第几章”“前两章”等章节范围，只写路线中已确认的教材和主题模块。
- 以连贯自然语言为主，保留下列少量栏目即可，不要输出表格、知识点 ID、系统字段或嵌套 JSON。

- 【当前主目标】写本周期一个可达成、可观察、可验收的目标。
- 【长期目标保温】写聚焦短期问题时维持长期主线的最低安排。
- 【具体任务块】写本周期内不同节点的学习范围、推进关系、阶段产出和验收标准；只需与今日任务方向一致，不要复制今日任务的逐步动作，也不要为了填满预算而增加任务。
- 【复习任务】只写本周期需要复习的对象、形式、产出和判定方式；实际到期任务由复习队列另行下发，不在短期计划正文中写日期或调度占位语。
- 【反馈指标】只写本周期能真实采集的完成率、正确率、遗漏项、错因和实际耗时，不虚构当前值。

## 边界

本 Skill 只提供中医药学习教育规划，不提供诊断、处方、疗效承诺或个体化诊疗建议。不得生成系统 ID、系统时间、持久化状态、知识点 ID、候选题参考结论、诊疗建议或教材证据之外的专业结论；临床高级能力必须保留导师或正式评价边界。
