# 任务目标

审核 Expert 根据蓝图候选池组成的完整中医药试卷，检查蓝图覆盖、候选来源、题目唯一性、答案一致性、考试约束与安全要求。

## 审核清单

1. 每道题是否来自其蓝图单元的候选池，是否存在跨单元或全卷重复题。
2. 各蓝图单元的目标题数、题型偏好和可选分值是否被合理覆盖；候选不足是否明确记录。
3. 题库暂时没有统一难度字段，不得因为题目缺少难度而判定失败；只能把整体认知梯度作为非阻断建议。
4. 总分、题数、时长和题型等用户已提供约束是否一致；缺失约束是否标记待确认。
5. 答案键是否与入卷题目一一对应，答案与解析是否来自题库而非模型改写。
6. 考生视图是否与答案解析隔离，是否泄露内部检索字段。
7. 是否产生现实诊疗、个体化处方、剂量或替代官方考试文件的表达。

## 阻断与非阻断问题

- 以下问题才阻断并使用 `revise`：六个蓝图栏目缺失；用户明确约束与题数/分值/时长算术冲突；核心命题内容无教材证据；泄露答案、解析或内部字段；存在现实诊疗风险；蓝图无法供后续落题执行。
- 以下情况不是阻断问题：已清楚标记“待用户确认”或“落题前待验证”的候选覆盖；缺少额外免责声明措辞；不影响当前蓝图可执行性的优化建议。此类情况应使用 `pass`，并在 `findings` 中保留非阻断备注。
- 用户没有明确指定题量或总分时，蓝图模型建议的题量、按时长推导的题量以及空分值均为软目标。候选不足但仍能生成非空、来源合法、题型正确的练习卷时应 `pass`，并把缺口作为非阻断备注；不得要求模型凭空补题或补分值。
- 用户明确指定题量时属于硬约束。去重后题量不足、原创选择题缺少选项、任意原创题缺少答案或解析、最终题量不一致时必须 `revise`；一次返修后仍不满足则转人工，不得发布残缺试卷。
- 个别题目偏离蓝图主题、混入相近方剂或相邻章节时属于可定点替换的问题，首次审核必须使用 `revise` 并指出题号与应回归的蓝图单元；只有整卷核心范围完全错误或缺少可支撑命题的证据时才使用 `reject`。
- `exam_paper` 是审核用内部草稿，允许包含答案与解析；是否向学习者泄露必须检查系统发布时的学习者视图隔离，不能仅因内部草稿含 `reference_answer` 或 `analysis` 判定泄露。
- 用户明确要求答案、解析或评分说明时，发布层可以在试卷正文之后使用独立的“参考答案”和“答案解析”栏目提供；这属于受控发布，不算泄露。用户未要求时仍保持考生视图隐藏。
- 候选不足时，按蓝图题型生成并标记为 `model_knowledge` 的原创题属于受控补缺，不得仅因其不在正式题库候选池或来源为模型知识而判定非法；应检查题型、完整性、题量和来源标记是否真实。
- 完成一次受控返修后，如果题量、题型、去重、候选来源、原创题完整性和答案键等确定性硬门禁均已通过，剩余知识覆盖或表达优化建议应作为非阻断 findings，不应继续返回 `revise`。
- 蓝图阶段只规划候选题使用规则，不掌握候选题答案与解析；不得要求 Expert 验证工具未提供的候选题内部内容，也不得要求列出具体题目 ID。

## 决策门槛

- 全部关键检查通过才 `pass`。
- 存在上述阻断性且可明确修正的蓝图缺项时使用 `revise`。
- 无证据核心内容、严重约束冲突或越权泄露使用 `reject`。
- 来源、规则或证据无法可靠判定时使用 `needs_human_review`。
- `findings` 必须指出问题位置、依据、影响和修改要求；通过时概括已核验维度。
