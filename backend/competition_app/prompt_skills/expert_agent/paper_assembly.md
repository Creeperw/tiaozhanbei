# 任务目标

依据正式试卷蓝图和按蓝图单元检索完成的候选题池，一次性选择题目并组成完整试卷草稿。

## 选题规则

1. 只能选择当前 `candidate_pool` 中存在的 `question_id`，且 `unit_id` 必须与候选所在单元一致。
2. 同一道题全卷只能出现一次。若跨单元重复，应选择最适合的单元并在覆盖摘要中说明缺口。候选题可以同时出现在多个单元池中，这是为了避免 Knowledge 阶段提前分配导致后续单元无题；最终入卷仍必须全局唯一。
3. 尽量满足每个单元的 `required_question_count` 和可选分值。用户明确题量且正式候选去重后不足时，必须在 `generated_items` 中只原创缺口数量，并严格匹配所属蓝图单元的题型；不得超量生成。每道原创题必须包含标准答案、解析、来源层级和所属单元；选择题必须包含至少两个选项，简答题等非选择题的 `options` 使用空列表。
4. 必须严格遵守蓝图单元的 `question_type_preferences`。选择题专项只能选择单项选择题和多项选择题；不得用判断、填空、简答或问答题凑数。
5. 当前题库没有可靠统一难度字段，不得要求候选必须有难度，也不得因难度缺失拒绝题目。可根据题干认知动作进行整体平衡，但必须作为选择理由而非客观难度事实。
6. 不修改题干、参考答案和解析；答案键与解析由系统从已选题目确定性派生。
7. `instructions` 是面向考生的卷面说明，不得泄露答案、解析、内部题目ID或检索信息。
8. `coverage_summary` 说明蓝图单元覆盖、题型分布、题数与分值加总，以及尚未解决的约束。
9. `external_question_references` 是网络搜索得到的外部题目线索，仅用于了解可能的题目方向；`selectable=false`，不得把其 `source_id` 当作正式 `question_id` 入卷。只有 `items` 中的正式题库题目可被选择。
10. 题型或主题不满足蓝图要求的题目不得用于凑数。精确候选不足时，先利用网络题目线索和教材证据确定命题方向，再在 `generated_items` 中生成带新 ID 的变式题；不得原地修改正式题库题目。
11. 原创题优先依据教材，其次使用有明确来源的网络参考；仍不足时允许使用模型自身通行知识，但 `source_tier` 必须标记为 `model_knowledge`，不得伪造教材或网络引用。
12. 用户未明确题型时遵循蓝图的均衡题型安排。软题量场景若某个单元没有可用正式候选，可在 `generated_items` 中为该空缺单元生成一道符合题型的题，确保综合试卷不退化为纯选择题；不得借此填满模型自行建议的全部软题量。
13. 每道题必须严格属于蓝图单元的 `retrieval_query` 和整卷 `scope_summary`。除非蓝图单元明确要求，不得把相近方剂、同类治法或同章节其他知识点当成目标主题。
