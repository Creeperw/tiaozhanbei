# 任务目标

在检索题目之前，根据用户组卷诉求和考试约束生成结构化试卷蓝图。每个蓝图单元必须能直接驱动后续题目检索。

## 蓝图单元要求

每个单元写明：知识模块、学习目标、检索表达、可选题型偏好、目标题数、可选分值、候选数量和选题规则。

- `difficulty_preference` 是可选偏好，不是必填字段。当前题库没有可靠统一的难度字段，难度不得成为检索硬过滤条件。
- `question_type_preferences` 允许为空；题型缺失时后续检索仍应执行。
- 用户明确要求“选择题专项”时，每个蓝图单元的 `question_type_preferences` 只能使用 `单项选择题`、`多项选择题`，不得加入判断题、填空题、简答题、问答题或病例问答。用户只说“选择题”但未限定单选/多选时，可同时允许单项与多项选择题。
- 只为用户本轮明确要求的知识范围建立单元。除非用户明确提出，不得自行增加相近方辨析、拓展方剂、其他治法或其他章节。
- `candidate_limit` 应大于目标题数，为整体组卷保留去重和筛选空间。
- 不生成系统 ID，不检索或选择具体题目，不生成试卷正文、答案或解析。
- 用户未提供题量、分值或时长时写入 assumptions 或保持对应可选字段为空，不得伪造官方规范。
- `session_time_budget_minutes` 是本次学习或操作的可用时间上限，不等于正式试卷时长；只有当前请求或 `exam_constraints.duration_minutes` 明确给出作答时长时才填写 `duration_minutes`。
- 用户未明确指定总题量时，各单元 `required_question_count` 只是候选池容量范围内的建议目标，不是发布硬门槛；不得根据分钟数推导“每分钟一题”等强制规则。用户明确写出题量时才作为硬约束。
- `planning_context` 中保留当前长期规划、短期计划和当日任务的模型相关内容；不得因为精简提示词而省略计划正文或结构化长期阶段。
- 用户引用“第N阶段”时，必须以 `learning_scope` 和 `planning_context.long_term_plan` 中的当前长期规划为组卷范围，不能被画像中的其他近期目标替换。结构化阶段存在时优先采用其目标和教材；只有正文时，从长期规划正文中的阶段路径和阶段里程碑解析，不得改用近期目标猜测该阶段。
- 用户未指定题型且试卷覆盖多个知识单元时，默认同时包含客观题与至少一种简答题或案例分析题；不得把综合测试卷自动收缩为纯选择题。

## 整体约束

当前自然语言请求代表用户本轮最终意图；当它明确给出的题量、题型、范围、时长或分值与 `exam_constraints`、用户画像等随附数据冲突时，以当前请求为准，随附数据仅补充当前请求未说明的约束。例如“全是填空题，至少25题”必须覆盖数据包中的“4题、单选与简答”配置，生成不少于25道且仅含填空题的蓝图；不得取交集、折中或沿用旧配置。

各单元的目标题数和分值应与用户明确约束一致。验收条件应覆盖候选来源、全卷去重、分值算术、题型结构、答案隔离和教学安全。
