# 题目训练提示工作台设计

## 目标

在保留训练工坊模块导航、题目加载、范围切换、作答、批改和训练产物流程的前提下，将“题目训练”改造成参考 Figma 的双栏工作台。移除原“证据 / 审核 / 轨迹”检查器，右栏改为按需展开的答题提示。

## 页面结构

- 训练工坊外层继续保留模块导航与训练产物。
- 题目训练顶部保留训练模式和题目范围切换。
- 主工作区左栏依次展示题目、知识点、提示触发器、答案输入和提交操作。
- 右栏为“答题提示”，默认锁定；用户点击“若暂时没有思路，点我查看提示”后展开。
- 提示只根据当前题型、难度、来源和知识点给出作答方法，不展示或伪造标准答案。
- 移动端改为单栏，提示面板排列在触发器之后、答案输入之前。

## 交互与状态

- 新题、题型、题目范围或知识点变化时，提示恢复锁定状态。
- 展开按钮使用 `aria-expanded` 和 `aria-controls` 关联提示面板。
- 提交、批改、下一题、错误和空状态沿用现有数据流。
- 原证据检查器按钮、移动端“证据”页签、抽屉、遮罩及键盘页签逻辑全部移除。

## 验证

- 组件测试覆盖提示默认锁定、点击展开和动态知识点内容。
- 页面测试确认只保留“任务 / 结果”移动页签且不存在证据检查器。
- 运行相关 Vitest、完整构建并进行桌面与移动视口截图检查。
