# 学习工坊阶段入口设计

## 目标

将“学习工坊”从复用学习路径的页面改为独立的学习阶段入口。用户先看到以 Figma 六阶段稿为视觉基线、由后端结果驱动的 4–6 阶段阶梯概览，点击阶段后进入现有学习路径页；学习路径页默认显示经典路线，只有用户主动选择时才切换到“我的学习路径”。

## 信息架构

学习工坊使用同一 `practice` 页意图下的三种视图：

- 无 `view` 或 `view=stages`：动态 4–6 阶段总览，作为学习工坊默认入口。
- `view=path`：现有学习路径页，默认为经典路线。
- `view=workspace`：现有题目训练、知识卡和试卷工作区。

这三种视图共用现有 `AppShell` 和 `onNavigate` 页意图机制，不引入新路由库。

## 阶段总览

新建独立的 `LearningStageLanding` 组件，按 Figma 节点 `63:219` 还原：

- 标题、副标题和与阶段数量一致的顶部阶段标记。
- 4–6 张逐步升高的渐变阶段卡片。
- 阶段名称、建议周期、核心任务和推荐资源。
- 底部进阶图例。

桌面端保留阶梯关系，卡片整体作为可键盘聚焦的按钮。屏幕宽度不足时改为从上到下的阶段列表，不使用水平缩放后的小字截图。

阶段内容先以经确认的六阶段配置作为产品基线，同时组件接收后端阶段数组。布局根据实际数量重新计算卡片宽度、间距与相对高度；准备 4、5、6 三种测试情形，不通过固定槽位隐藏或补造阶段。阶段状态不伪造用户进度；在后端没有稳定的阶段进度契约前，只展示内容导览。

## 交互与转场

- 阶段卡片借用 OriginKit `Magnetic Carousel` 的距离衰减机制：桌面精细指针靠近时，当前卡片抬升和放大，相邻卡片按距离递减响应；触控设备不启用磁吸。
- 点击或键盘激活卡片后，在应用路由层上方生成持续存在的卡片副本，避免页面切换导致动画中断。
- 转场借用 OriginKit `Flip Gallery` 的透视、正反面和 `rotateY(0deg → 180deg)` 结构。翻转中点切换到 `view=path`，动画结束后才移除覆盖层。
- 翻转背面显示阶段名称和“经典路线”，动画期间拦截重复操作。
- 使用 `framer-motion` 承载翻转与几何位移动画；`prefers-reduced-motion` 下改用约 80ms 的无旋转切换。

## 卡片内部布局

- 编号、标题、周期和任务列表在所有卡片中使用一致的顶部节奏，任务区起始线不随卡片高度漂移。
- 卡片高度采用“足够容纳核心内容的基础高度 + 按阶段进度等差递增”的视口公式，不能用统一最大高度截断后两级的阶梯差。
- 核心任务始终完整展示，不允许内部滚动，也不依赖 `overflow: hidden` 裁掉文字。
- 进度位于前半段的较矮卡片使用单行紧凑资源行；后半段卡片显示带标签的完整资源区。两种资源区都位于卡片底部且不与任务重叠。
- 删除占据文档流的“查看学习路径”提示，整张语义化按钮已经表达可点击性；鼠标悬停和键盘聚焦继续提供视觉反馈。
- 默认数据和未来后端数据共用同一密度规则，密度由阶段在当前 4–6 项列表中的归一化位置决定，不写死第几张卡片。

## 阶段线稿素材

- 六张用户提供的竖版线稿按“基础筑基、经典研读、中药方剂、临床实践、专科深化、融会贯通”映射到默认阶段。
- 素材作为装饰性水印，使用空 `alt`、`aria-hidden` 和 `pointer-events: none`，不重复朗读卡片语义，也不影响点击。
- 图片层位于渐变背景与文字内容之间，文字、勾选项和资源标签始终保持更高层级。
- 原始 PNG 没有透明通道且包含浅色棋盘格，因此通过低透明度与 `mix-blend-mode: multiply` 融合，不直接以不透明矩形展示。
- 第 2 与第 6 张源文件内容相同：经典研读聚焦下部的书籍循环，融会贯通聚焦右上人体与上升路径，以不同裁切降低重复感。
- 后端返回自定义阶段时允许提供可选 `illustration` 与焦点参数；没有图片的阶段仍正常显示渐变卡片。

## 导航与阶段上下文

点击阶段后发送：

```js
onNavigate({
  page: 'practice',
  params: {
    view: 'path',
    pathMode: 'classic',
    stageId: stage.id,
    stageIndex: index,
  },
});
```

学习路径页保留阶段上下文。如果经典路线数据能稳定匹配该阶段，页面将其作为初始聚焦；如果不能匹配，仍展示完整经典路线，不阻断用户。

学习路径页增加“返回学习阶段”操作。顶部训练模块、今日安排、智能助教、路线选择器和现有路径钻取能力保持不变。

## 路径默认值

`DashboardPage` 作为 `view=path` 的学习路径页时：

- 默认 `pathMode='classic'`。
- 经典路线列表加载后默认选中第一条可用路线。
- 用户点击“我的学习路径”后才展示个性化规划数据。
- 用户没有长期规划时，经典路线仍可使用；只在切换到个性化路径时显示规划引导。

## 错误和空状态

- 后端阶段结果尚未接入或为空时使用六阶段基线数据，入口始终可用。
- 经典路线加载失败时使用现有空状态和错误文案。
- 阶段与经典路线无法匹配时回退到路线根视图。
- `view=workspace` 深链和指定 `taskType` 的跳转不经过阶段总览。

## 验收与测试

- 从侧边栏点击“学习工坊”显示后端决定的 4–6 阶段总览。
- 4、5、6 阶段均能自动调整间距和高度，不发生横向溢出。
- 点击任一阶段先完成磁吸响应和跨页面翻转，再进入学习路径页。
- 学习路径页初始选中“经典路线”。
- 点击“我的学习路径”正常切换，不影响经典路线。
- “返回学习阶段”返回阶段总览。
- 题目训练、知识卡和试卷的现有 `view=workspace` 深链保持有效。
- 键盘用户可遍历和激活全部阶段卡片，移动端无水平溢出，减少动态效果设置生效。
