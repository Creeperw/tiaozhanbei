# 学习工坊阶段悬浮卡片与自然高度设计

## 背景

新版学习工坊已经将学习路径作为默认一级页面，并在右上角提供当前阶段入口。实际页面中，阶段简略卡片弹层受到学习路径容器的 `overflow: hidden` 裁切，导致底部“查看完整进阶路线”入口不可见。同时学习工坊仍继承全屏工作区高度，昨日学习反馈下方出现较大无效空白。

## 目标

1. 鼠标悬浮或键盘聚焦右上角当前阶段时，完整展示阶段简略卡片。
2. “查看完整进阶路线”始终可见，点击后继续进入现有完整阶段页面。
3. 学习路径画布仍保持自身裁切，不让节点或连线溢出卡片边界。
4. 桌面学习工坊按内容自然收高，在昨日学习反馈后结束。
5. 保留移动端模态底部阶段面板及现有焦点管理。

## 方案选择

采用“调整裁切边界 + 内容自然高度”。

- 学习路径外层允许阶段弹层溢出。
- 路径画布 `.dashboard-daily__path-stage` 继续负责裁切路径内容，并补齐下方圆角。
- 阶段弹层保持右对齐的绝对定位，不引入 Portal。
- 缩短触发器与弹层间距，使鼠标能稳定从阶段牌组移入弹层。
- 弹层设置合理的视口最大高度；完整路线按钮固定在弹层内容末端并保持可见。
- 仅在桌面和平板布局取消训练工坊的强制全屏高度；移动端继续使用当前自然滚动布局。

不采用 Portal，因为它会引入触发器定位、滚动同步和窗口变化监听；不采用布局内展开，因为会推动路径图和训练模块，失去悬浮卡片的交互感。

## 交互

### 桌面与支持悬浮的设备

- 鼠标进入当前阶段牌组，弹出简略阶段卡片。
- 鼠标可直接移入弹层操作，不因触发器与弹层之间的间隙关闭。
- 当前阶段卡片置顶，其余阶段维持网格排列和完成、当前、待解锁状态。
- 点击任一阶段沿用当前阶段选择逻辑。
- 点击“查看完整进阶路线”导航到 `{ page: 'practice', params: { view: 'stages' } }`，复用现有完整阶段页和过渡动画。
- 鼠标离开整个阶段选择区域或焦点移出后关闭。

### 移动端

- 点击当前阶段继续打开现有模态底部面板。
- 保留遮罩、滚动锁、焦点进入与循环、关闭后焦点恢复。

## 布局

- 桌面训练工坊取消 `height: 100%`、`min-height: 100dvh` 和填满剩余空间的伸展行为。
- 主工作区高度由学习路径、训练工坊、昨日反馈和右侧工作栏的实际内容共同决定。
- 页面底部 padding 收紧，反馈卡片后只保留正常视觉间距。
- 路径卡片继续使用既定的紧凑高度，不通过重新放大路径图消除空白。

## 验证

1. 单元测试先验证悬浮可打开阶段弹层。
2. 验证弹层包含可见的“查看完整进阶路线”，点击后导航到完整阶段页。
3. 回归移动端模态、焦点管理和阶段选择测试。
4. 运行学习工坊关联测试、ESLint 和生产构建。
5. 在桌面、平板和移动宽度检查：
   - 弹层不被路径卡片裁切；
   - 完整路线按钮位于视口内；
   - 路径画布仍被正确裁切；
   - 昨日反馈后不存在可滚动的额外空白区域。
